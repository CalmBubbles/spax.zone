ActorProxy¶

Introduced in NotITG v1

  • Description

  • API reference

Description¶

An ActorProxy is used to tell an actor to draw itself again. The actor to be drawn is its “target”, and can be set by calling ActorProxy.SetTarget(). Note that an ActorProxy will draw even if its target is hidden.

Any shaders set on the targetted actor will continue to be used when redrawn by the ActorProxy - you should not call Actor.SetShader() on the ActorProxy object itself.

Some practical uses for an ActorProxy include…

  • Making more playfields appear (An ActorProxy should be preferred over enabling playfields 3 through 8 since proxies are more lightweight)

  • Changing the draw order of actors you can’t modify (Since actors cannot be moved between background, playfield, foreground, etc. layers, you can hide the original actors and proxy them instead to achieve the same effect)

  • Moving judgment sprites/combo text independently of the playfields (This can be helpful if you’re moving the playfield all over the place and don’t want the judgment/combo actors following along)
