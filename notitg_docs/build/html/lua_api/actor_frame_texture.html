ActorFrameTexture¶

Introduced in NotITG v1

  • Description

  • API reference

Description¶

Often called a render target in other applications, an ActorFrameTexture (AFT) captures everything that was rendered before it into a texture.

An example of an ActorFrameTexture, rendered onto many sprites, taken from [WinDEU] Camellia - Maze of Vignere Square

Note that this does not function like the ActorFrameTexture from StepMania 5! StepMania 5’s ActorFrameTexture captures all of its children into a texture, whereas NotITG’s one captures everything rendered before it.

Some example uses of ActorFrameTextures include:

  • Post processing effects (such as bloom - achieved with a ActorFrameTexture to capture a framebuffer, and a Sprite to render the framebuffer with a custom shader program)

  • Hazey appearences (like the above)

  • Making smears by reusing textures from previous frame, or by not clearing the ActorFrameTexture between frames

EnablePreserveTexture(enable: boolean) → void¶

Sets whether the render target should be cleared before drawing to it

If preserve texture is enabled, then the render target will not be cleared when rendering.

This is disabled by default (ie: it defaults to clearing the target before rendering)

Introduced in NotITG v1
